# PTAG Public Opening Announcement Draft

## Short Version

PTAG is now public in the SA-NOM repository.

From this point forward, PTAG should no longer be read here as a proprietary framework with a half-exposed public story. The repository now presents PTAG as a public governance-language layer with documentation, reference implementation paths, public-safe examples, and a contribution path that others can study and improve.

Repository:
- https://github.com/sanom-ai/sa-nom-ai-governance-suite

PTAG framework guide:
- https://github.com/sanom-ai/sa-nom-ai-governance-suite/blob/main/docs/PTAG_FRAMEWORK.md

## Long Version

PTAG started inside SA-NOM as the policy and role-governance layer that helps define roles, authority boundaries, constraints, and policy logic in a structured, reviewable form.

Today, that PTAG story becomes public in a clearer and more complete way.

What changed:
- PTAG is now described publicly in the repository rather than framed as a partially hidden internal layer
- the repository points directly to PTAG implementation paths and public-safe examples
- the PTAG framework guide now explains public status, headers, block structure, validation posture, syntax position, and repository reference files more explicitly
- contributors now have a clearer path for PTAG documentation, parser, validator, semantic, and example changes

What this means:
- you can study PTAG directly from the repository
- you can use the public materials as a basis for your own governed-role and policy experiments
- you can contribute improvements through the normal repository workflow

What this does not mean:
- PTAG is not being claimed as a frozen standard
- the public opening does not eliminate the need for human judgment, governance review, or controlled deployment decisions
- the language can continue to evolve in public

If you want to start reading PTAG now:
1. Read `docs/PTAG_FRAMEWORK.md`
2. Read `docs/PTAG_FULL_SPEC.md`
3. Inspect `resources/roles/`
4. Inspect `sa_nom_governance/ptag/`
5. Read `CONTRIBUTING.md` if you want to help improve it
