# ROADMAP v0.7.8 - Private-First Tablet Command Surface

Status: In progress  
Line: v0.7.8-tablet-command-surface

## Goal
Make SA-NOM feel native on organization-managed tablets while preserving the private-first command-surface model: one human governs the organization, AI performs most of the work, and deeper governance tooling stays behind protected surfaces.

## Slices

### 1. Tablet Command Surface Foundation
- tablet-first shell for the simple Home dashboard
- touch-safe navigation and command actions
- compact command dock instead of a desktop-only left rail
- quick access cards that stay readable on tablet landscape and portrait
- progressive disclosure stays intact

### 2. Private Session Continuity
- clearer reconnect and session continuity posture on tablet devices
- resilient refresh, idle, and sign-out flow for organization-managed screens
- preserve the private local runtime story without adding cloud assumptions

### 3. Role-Based Tablet Views
- executive, department-head, and operator personas stay on one shell
- persona differences show as focus, not as separate applications
- normal users keep the simple command surface while advanced governance stays protected

### 4. Touch-First Work Flows
- approvals, details, document review, AI actions, and case follow-through stay thumb-friendly
- command surface keeps answering posture, next action, and AI activity inside one tablet rhythm

## Slice 1 Definition of Done
- the command surface no longer feels like a desktop layout compressed onto a tablet
- the main Home experience works cleanly in tablet landscape and remains readable in portrait
- top navigation, organization switcher, and primary actions are touch-safe
- the command dock becomes compact and secondary so Home answers the next move first
- quick-access department cards prioritize real work context instead of static ordering
