# Issue Drafts - v0.1.15

Use these draft issues to turn the `v0.1.15` roadmap into implementation work.

## 1. Add Security Ownership And Labels Guide

**Goal**
Document how security exceptions, follow-up items, and escalated findings should be owned and labeled.

**Scope ideas**
- ownership roles for follow-up and escalation
- label vocabulary for security workflow
- plain-language examples for issue triage

**Success signal**
Security workflow becomes easier to inspect and less dependent on ad hoc maintainer memory.

## 2. Refine Security Issue Templates For Ownership Clarity

**Goal**
Make the existing security issue templates separate AI-preparable context from human decision fields more clearly.

**Scope ideas**
- distinguish summary fields from human approval fields
- reinforce escalation owner and revisit owner fields
- tighten language around who confirms what

**Success signal**
The issue templates better reflect SA-NOM's AI-heavy but human-gated operating model.

## 3. Evaluate Security Issue Forms

**Goal**
Decide whether SA-NOM should keep markdown issue templates or move toward structured GitHub issue forms for security workflow.

**Scope ideas**
- form fields that AI can prepare reliably
- fields that must remain human-confirmed
- migration costs versus workflow benefit

**Success signal**
The repository has a clearer direction for how structured its security issue intake should become.

## 4. Align Security Workflow Docs With Ownership Rules
