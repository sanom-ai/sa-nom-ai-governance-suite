# Issue Drafts - v0.5.2

Use these issue drafts to deliver the `v0.5.2` patch-and-polish phase.

## 1. Polish The Private Dashboard Trial Flow

**Goal**
Reduce small operator-facing friction in the private dashboard without changing the overall runtime model.

**Scope ideas**
- tighten first-run onboarding copy and handoff cues
- improve navigation between overview, requests, overrides, Human Ask, health, and audit surfaces
- refine labels, emphasis, and guidance where evaluators hesitate during live trials

**Success signal**
The dashboard feels smoother and more self-explanatory during real evaluation use.

## 2. Improve The Evaluation Path For New Users

**Goal**
Make the repository easier to try without requiring as much owner-guided explanation.

**Scope ideas**
- strengthen guided-evaluation paths and cross-links
- clarify when to use quick start, private dashboard mode, or other evaluation surfaces
- reduce confusion around tokens, local evaluation, and first-run expectations

**Success signal**
A new evaluator can choose a sensible trial path more quickly and with less ambiguity.

## 3. Fix Patch-Level UX Gaps And Trial-Use Bugs

**Goal**
Close small gaps discovered during hands-on use so the serious-evaluation surface feels tighter.

**Scope ideas**
- wording mismatches
- light dashboard or docs polish
- low-risk runtime or presentation bug fixes uncovered during manual trials

**Success signal**
`v0.5.2` removes obvious rough edges without growing into a new concept phase.

## 4. Close v0.5.2 As A Small Hardening Release

**Goal**
Finish the patch line with a release story that clearly explains what was tightened and what remains for the next major phase.

**Scope ideas**
- release notes for `v0.5.2`
- concise explanation that this is a polish-and-hardening patch
- explicit separation between patch work and the future `v0.6.0` concept line

**Success signal**
`v0.5.2` reads as a clean, disciplined patch release rather than a hidden scope expansion.
