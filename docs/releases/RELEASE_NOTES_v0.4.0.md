# Release Notes - v0.4.0

## Release Focus

`v0.4.0` is the PTAG public-opening release after the `v0.3.0` usability phase.

This release is focused on making PTAG legible as a public governance-language layer by aligning repository language, framework documentation, contribution guidance, and public-facing release framing with the PTAG materials already present in the codebase.

## Completed Scope

### PTAG Public Positioning

- removed outdated proprietary-language framing from the public PTAG story
- aligned README and framework-language positioning with the repository's public license posture
- reframed PTAG as a public repository surface rather than a partially disclosed internal concept

### PTAG Specification Hardening

- expanded PTAG documentation around headers, block types, validation model, syntax position, and example structure
- added a separate PTAG full spec and connected public PTAG docs to the parser, validator, semantic, and role-resource files already present in the repository
- clarified what the current public PTAG surface does and does not claim yet

### PTAG Contribution Path

- added PTAG-specific contribution guidance for syntax, validator, semantic, and example changes
- made public expectations clearer for documentation and test updates when PTAG behavior changes

### Planning And Public Opening Package

- added `v0.4.0` roadmap and issue-draft materials for the PTAG public-opening phase
- prepared repository-facing release and announcement materials for the PTAG opening story

## Why This Release Matters

`v0.4.0` changes PTAG from a mostly product-described concept into a more credible public repository surface.

The repository now gives readers a clearer path to:
- understand what PTAG is
- inspect the current reference implementation
- study public-safe role examples
- reason about validation and policy-coverage posture
- contribute improvements through the normal repository workflow

## Public Opening Note

This release should be read as a public-opening milestone, not as a claim that PTAG is already a frozen standard.

The intent is to make PTAG open enough to study, use, evaluate, and improve in public while still being honest about the fact that the language and its tooling can continue to mature.

## Next Phase

The next milestone after `v0.4.0` should continue PTAG hardening through richer examples, stronger validation depth, better tooling support, and clearer external comparison materials, while keeping the public claim disciplined and evidence-based.

## Notes

- this release closes the first PTAG public-opening phase
- PTAG is now described publicly in line with the repository artifacts available to readers
- public positioning was tightened without claiming that every future PTAG design decision is already settled
