# Release Notes - v0.6.0

## Release Focus

`v0.6.0` is a major-direction release that opens the next strategic phase of SA-NOM after the `v0.5.x` patch line.

This release does not present a finished implementation of global cultural or regulatory alignment.
Instead, it establishes the official direction, architecture, safe claims, non-goals, and first controlled implementation path for:

`SA-NOM Global Harmony Protocol`

The intent is to move SA-NOM from governed operations alone toward governed global-local alignment without turning the system into a fragmented set of country-specific products.

## Completed Scope

### Global Harmony Protocol Roadmap

- added the `v0.6.0` roadmap for a structure-first global-local alignment phase
- defined the major-phase goal as one coherent alignment direction rather than a broad feature explosion
- positioned `v0.6.0` as the architectural opening for the Global Harmony Protocol line

### Safe Claims, Boundaries, And Non-Goals

- defined public-safe claims for discussing regional cultural and regulatory alignment in a governed SA-NOM context
- made it explicit that `v0.6.0` does not claim complete global coverage, automatic legal compliance, or replacement of local experts
- created a cleaner boundary between strong enterprise positioning and careless overclaiming

### Global Harmony Protocol Concept Paper

- added a dedicated concept paper for `SA-NOM Global Harmony Protocol`
- described the `Single Alignment Engine` idea as the core structural principle
- connected regional constitutions, regulatory resonance, linguistic and cultural authenticity, and dynamic alignment switching back to PTAG, Human Override, and Audit Trail

### First Controlled Implementation Path

- positioned `v0.6.1` as the first narrow implementation slice rather than implying full readiness in `v0.6.0`
- identified regional constitution templates, baseline loading, dashboard alignment controls, and governed alignment inputs as likely first-step candidates
- preserved a believable path from concept to implementation without collapsing planning and delivery into the same release

## Why This Release Matters

`v0.6.0` matters because it turns a promising idea into an official major-phase direction with disciplined boundaries.

The repository now gives readers a clearer way to understand that SA-NOM is preparing for a future where governed AI roles may align not only with universal safety and operational control, but also with regional cultural posture and regulatory expectations.

Just as importantly, the repo now states what this phase is not:
- not a claim of worldwide readiness
- not a substitute for local legal or cultural expertise
- not a finished implementation of resonance scoring or global alignment automation

That positioning makes the direction stronger, not weaker, because it keeps trust aligned with what the repository can actually support today.

## Verification

- planning materials were added through the normal PR workflow
- `v0.6.0` is intentionally a concept-and-architecture release rather than an implementation release

## Next Phase

The next step after `v0.6.0` should be `v0.6.1` as the first controlled implementation slice of the Global Harmony Protocol direction.

That phase should stay narrow and credible, focusing on baseline alignment loading, initial constitution templates, and the smallest useful control surface needed to demonstrate the concept in governed form.

## Notes

- `v0.6.0` is best read as a major-phase opening for the Global Harmony Protocol line
- this release formalizes direction and architecture first so later implementation can proceed without narrative confusion
- the repository now has a clearer bridge between governed operations and future governed global-local alignment work
