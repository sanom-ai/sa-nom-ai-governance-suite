# Release Notes - v0.1.4

## Release Focus

This release sharpens SA-NOM's public commercial story so evaluators, buyers, and internal champions can move from repo interest into a clearer demo, qualification, ROI, and pilot conversation.

## Highlights

- Expanded the public feature matrix into a more detailed community-versus-commercial guide
- Strengthened the commercial discovery checklist with ROI framing, pilot fit, success criteria, and clearer next-step outputs
- Added a one-page demo checklist for live customer walkthroughs
- Added a one-page ROI framing document for budget, pilot, and executive conversations
- Linked the new commercial and demo assets through the main README and docs index

## Community Baseline In This Release

- clearer public explanation of what is included in the self-managed community baseline
- stronger guidance for how Ollama-first private evaluation connects to demo and rollout conversations
- a faster path for preparing live demos without reconstructing the story from multiple docs
- public-safe ROI framing that supports customer conversations without overclaiming guaranteed savings

## Commercial Direction

Commercial offerings remain separate from the AGPL community baseline and may include rollout support, enterprise controls, tailored compliance programs, air-gapped delivery, and custom integrations.

See [../FEATURE_MATRIX.md](../FEATURE_MATRIX.md), [../COMMERCIAL_LICENSE.md](../COMMERCIAL_LICENSE.md), [../COMMERCIAL_DISCOVERY_CHECKLIST.md](../COMMERCIAL_DISCOVERY_CHECKLIST.md), and [../ROI_ONE_PAGER.md](../ROI_ONE_PAGER.md) for the current commercial boundary and sales path.

## Upgrade Notes

- this release is documentation and commercial-enablement focused; runtime behavior is unchanged
- `docs/FEATURE_MATRIX.md` now gives a much more detailed public boundary between community capabilities and guided or commercial paths
- `docs/COMMERCIAL_DISCOVERY_CHECKLIST.md` now includes ROI framing, pilot fit, success criteria, and anti-pattern guidance
- `docs/DEMO_CHECKLIST.md` can be used as a one-page operator cheat sheet during live calls
- `docs/ROI_ONE_PAGER.md` can be used as an executive-facing business-case support document during evaluation and pilot conversations
