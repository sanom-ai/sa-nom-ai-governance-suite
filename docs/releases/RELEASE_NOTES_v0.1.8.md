# Release Notes v0.1.8

## Release Focus

This release surfaces SA-NOM's embedded governance and structural-intelligence frameworks more clearly in the public product story.

The main goal of `v0.1.8` is not to introduce a new runtime engine, but to make the platform's foundational language and structural-assessment layers easier to understand for operators, buyers, and evaluators.

## Highlights

- surfaced PT-OSS more clearly as SA-NOM's embedded structural intelligence layer
- clarified that PT-OSS is a proprietary framework developed by the creator and integrated into SA-NOM
- added a plain-language PT-OSS metric explainer for `HDI-S`, `HDI-D`, `SFS`, `KPIR`, `ASP`, `HOIS`, and `SPAI`
- surfaced PTAG more clearly as SA-NOM's policy and role-governance language
- clarified that PTAG is a proprietary framework developed by the creator and integrated into SA-NOM
- updated the public README, product tour, feature matrix, and docs index so the PTAG and PT-OSS layers are easier to discover and explain

## Community Baseline In This Release

- clearer public explanation of PTAG as the role, authority, constraint, and policy-language layer inside SA-NOM
- clearer public explanation of PT-OSS as the structural intelligence layer used to assess dependency, fragility, override integrity, and asymmetry posture
- easier-to-use metric vocabulary for explaining PT-OSS in demos, buyer conversations, and internal evaluation
- stronger product-level visibility for creator-developed proprietary frameworks already embedded in the platform

## Upgrade Notes

- `v0.1.8` is a product-story and documentation milestone, not a runtime-behavior milestone
- PTAG should now be read publicly as SA-NOM's policy and role-governance language
- PT-OSS should now be read publicly as SA-NOM's structural intelligence layer
- the PT-OSS metrics now have a dedicated plain-language explainer to make the acronyms easier to understand
- the release strengthens how SA-NOM communicates its embedded frameworks without changing the underlying governance engine behavior
