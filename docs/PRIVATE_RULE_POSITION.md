# Private Rule Position (Historical Note)

This document is retained as a historical note because older SA-NOM materials used the terms `Private Rule Studio` and `Private Rule Position`.

Those terms should now be read as earlier concept language.
The current canonical product name is [`Role Private Studio`](ROLE_PRIVATE_STUDIO.md).

## Current Reading

What older notes called:

- `Private Rule Studio`
- `Private Rule Position`

should now be read as:

- `Role Private Studio`
- organization-defined governed hats created from normal JD-style inputs

The platform direction did not change.
What changed is the public-facing naming, so the repository now stays centered on one clear concept.

## Why The Name Changed

The current name is clearer because SA-NOM is centered on governed AI roles, role publication, and role-based operational work.

`Role Private Studio` more accurately reflects that:

- users provide normal role or operating inputs
- SA-NOM transforms them into PTAG-backed governed role packs
- the resulting hats become part of the governed runtime, not a separate rule-only subsystem
