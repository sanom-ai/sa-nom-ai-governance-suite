# Private Rule Studio And Private Rule Position

`Private Rule Studio` is a core SA-NOM capability.

It is the place where an organization can upload, adapt, or define its own internal job descriptions, operating rules, and role boundaries before turning them into governed AI roles.

A `Private Rule Position` is the resulting organization-specific hat or position that the organization defines for itself.

This position does not need to match a standard role pack or an official org-chart title. It can be a temporary operating hat, a cross-functional seat, or a narrow escalation role that only exists inside that organization's workflow.

## What This Means In Practice

With Private Rule Studio, an organization can:

- upload a JD or internal operating brief
- start from a starter JD, starter rule set, or starter hat prepared by SA-NOM
- create a custom `Private Rule Position`
- define authority boundaries, wait-human actions, and forbidden actions
- publish the result into governed runtime behavior through the existing studio and registry flow

## Private Rule Position Examples

Examples of a `Private Rule Position` include:

- `Factory Recovery Coordinator`
- `Customer Claim War Room Lead`
- `Launch Freeze Exception Coordinator`
- `Group Procurement Policy Escalation Lead`
- `Bank Regulator Liaison Lead`

These may not exist as permanent job titles in an official HR system, but they still represent real work with real authority boundaries.

## Core Capability, Not A Paid Unlock

SA-NOM should not be read as if customers must buy separate permission to create rules.

`Private Rule Studio` is part of the core platform story.

Organizations should be able to:

- create their own positions
- upload or edit their own JD inputs
- start from provided starter materials
- adapt those materials to fit their own context

The commercial layer is not about placing an artificial limit on rule creation.

## What SA-NOM Provides As Accelerator Support

SA-NOM can provide starter support in the form of:

- starter JD examples
- starter rules
- starter hats
- public-safe role packs and scenarios
- guided examples that show how to structure authority and escalation

Organizations can use these directly or customize them in Studio.

## What Commercial Tailoring Covers

Commercial work should focus on:

- organization-specific rule design
- industry-specific starter libraries
- workshops for authority and escalation modeling
- rollout support and publication governance
- integration of Private Rule Positions into real operating programs

This keeps the product model clean:

- core capability stays in the platform
- starter content accelerates adoption
- tailoring and rollout become the commercial service layer

## Relationship To Role Private Studio

A simple way to read the model is:

1. `Private Rule Studio` helps define the organization-specific role logic
2. `Private Rule Position` is the flexible hat the organization wants to create
3. `Role Private Studio` takes that governed position into publishable runtime form

That means SA-NOM is not limited to standard functions like Legal, HR, Finance, or QA. It can also support custom positions that only make sense inside one organization's real operating model.
