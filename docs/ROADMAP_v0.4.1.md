# Roadmap - v0.4.1

This roadmap opens the `v0.4.1` follow-on phase after `v0.4.0` publicly opened PTAG in the repository.

## Theme

Strengthen the PTAG public surface without expanding scope too broadly.

## Milestone Goal

By the end of `v0.4.1`, PTAG should feel easier to start with, easier to compare, better supported by examples, and more credible as a public governance-language baseline.

## Core Position

`v0.4.1` is the PTAG polish-and-hardening phase.

It should not try to reinvent PTAG or reopen the public-positioning decision that `v0.4.0` already established. Instead, it should make the current public PTAG surface easier to adopt, easier to evaluate, and safer to evolve.

## Workstreams

### 1. PTAG Quick Start

Target areas:
- a short `How to start with PTAG` path for new readers
- the smallest useful reading and example path
- clearer entry points from README and docs

Expected outcomes:
- new readers can understand where to start in minutes
- PTAG feels approachable rather than dense
- the public opening from `v0.4.0` becomes easier to use immediately

### 2. PTAG Examples Expansion

Target areas:
- more PTAG examples from simple to enterprise-shaped
- clearer examples for roles, authorities, constraints, and policies
- examples that help readers move from syntax to intent

Expected outcomes:
- PTAG examples become more reusable for real experiments
- the repo gives a better bridge from concept to implementation
- contributors have better public-safe reference material

### 3. PTAG FAQ And Comparison Surface

Target areas:
- a FAQ for common PTAG questions
- public comparison notes against adjacent tools or concepts where helpful
- explanation of what PTAG is and is not trying to replace

Expected outcomes:
- maintainers spend less time repeating the same explanation manually
- readers can place PTAG in the broader landscape more quickly
- outside evaluation becomes easier and more honest

### 4. PTAG Validation And Test Hardening

Target areas:
- stronger tests for PTAG syntax and semantic behavior
- clearer coverage around policy-coverage-gap and role-authority consistency paths
- tighter confidence that docs still match implementation behavior

Expected outcomes:
- PTAG changes become safer to review
- the public technical story is backed by stronger verification
- future syntax or tooling changes have a firmer baseline

## Non-Goals

- no major syntax migration in `v0.4.1`
- no premature declaration that YAML is now the primary PTAG format
- no broad runtime-architecture rewrite just because PTAG is now public
- no attempt to solve every comparison or standards question in one release

## Candidate Deliverables

- `How to start with PTAG` quick-start doc
- expanded PTAG examples
- PTAG FAQ and comparison doc
- PTAG-focused test additions
- `v0.4.1` release notes and closeout summary

## Exit Criteria

- PTAG has a fast entry path for new readers
- the repo includes more public-safe PTAG examples
- maintainers have a reusable FAQ/comparison surface
- PTAG syntax and semantic behavior are covered by stronger tests
- `v0.4.1` makes `v0.4.0` feel sturdier without changing its public contract
