# Feature Matrix

This matrix describes the intended open-core boundary for the public SA-NOM repository.
It is a product-positioning document, not a binding commercial contract.

| Area | Community Baseline | Commercial Layer |
| --- | --- | --- |
| PTAG parsing and validation | Included | Extended policy packs, rollout guidance, and tailored modeling support |
| Core governed runtime | Included | Organization-specific policy design, hardening, and delivery services |
| Dashboard and local operations | Included | Production rollout support, custom operational playbooks, and response commitments |
| Role Private Studio | Included | Commercial templates, review workflow extensions, and assisted publication programs |
| Human Ask workflow | Included | Enterprise routing, managed escalation directories, and program-specific operating models |
| Trusted Registry and readiness workflows | Included baseline path | Hardened deployment patterns, air-gapped delivery, and regulated rollout support |
| Model providers | OpenAI, Claude, and Ollama configuration plus runtime probes | Managed connectors, provider governance workshops, and tailored deployment patterns |
| Kubernetes packaging | Starter Helm chart and raw manifests | Cluster rollout services, hardened overlays, and environment-specific tuning |
| Integrations | Baseline hooks and examples | SSO/SCIM, commercial connectors, private APIs, SIEM/SOAR, and custom adapters |
| Support model | Self-managed community baseline | Direct support, training, rollout services, and negotiated response targets |
| Compliance and sovereign delivery | Public baseline examples, security checklist, and Thai starter templates | Tailored compliance mapping, sovereign packaging, and on-site enablement |
