# Roadmap - v0.1.1

This roadmap captures the highest-value follow-up work after the first public `v0.1.0` release.

## Theme

Stabilize the community baseline, improve contributor onboarding, and make the commercial path easier to evaluate without weakening the open-core boundary.

## Technical Priorities

- Run and stabilize the full `pytest` suite in a clean environment with `requirements-dev.txt` installed.
- Reduce environment friction for first-time evaluators.
- Add package metadata or a simple `pyproject.toml` if the project is going to support cleaner installation paths.
- Continue hardening access-control, deployment-readiness, and trusted-registry workflows.

## Product and DX Priorities

- Improve dashboard and landing copy so evaluators understand the product model faster.
- Expand docs for self-managed deployment, troubleshooting, and common operator flows.
- Add more public-safe examples that show real use without leaking private runtime state.
- Publish clearer guidance for community vs commercial adoption decisions.

## Community Priorities

- Triage the first public issues and label them clearly as community, commercial, docs, or roadmap.
- Tighten contribution flow once external pull requests start arriving.
- Add small good-first-issue items once the initial public feedback settles.

## Commercial Priorities

- Turn the sales intake template into a repeatable intake workflow.
- Prepare a short standard demo path for discovery calls.
- Define the first post-launch commercial package boundaries more explicitly.

## Exit Criteria For v0.1.1

- Full regression suite runs cleanly in a documented environment.
- README and deployment docs feel polished for first-time evaluators.
- Community vs commercial boundary remains clear after the first wave of feedback.
- At least one post-launch refinement cycle lands in the public repo.
